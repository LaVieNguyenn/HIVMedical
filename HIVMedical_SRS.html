1. Introduction

1.1 Purpose

This document specifies the software requirements for the HIVMedical System, a comprehensive healthcare management platform designed specifically for HIV patient care. The system aims to improve medication adherence, streamline appointment scheduling, and enhance patient-doctor communication.

1.2 Document Conventions

  • Functional Requirements: Prefixed with FR-
  • Non-Functional Requirements: Prefixed with NFR-
  • Business Rules: Prefixed with BR-
  • User Stories: Prefixed with US-

1.3 Intended Audience

  • Development Team
  • Project Managers
  • Healthcare Professionals
  • System Administrators
  • Quality Assurance Team

1.4 Product Scope

The HIVMedical System is a web-based application that provides:

  • Patient registration and profile management
  • Medication adherence tracking
  • Appointment scheduling
  • Medical record management
  • Automated notification system
  • Role-based access control

2. Overall Description

2.1 Product Perspective

The HIVMedical System is a new, self-contained system that integrates multiple healthcare management functionalities into a unified platform. It consists of:

  • Authentication Service: User management and authorization
  • Patient Service: Core patient management functionality
  • API Gateway: Centralized API management
  • Notification Service: Automated reminders and alerts

2.2 User Classes and Characteristics

2.2.1 Patients

  • Primary users of the system
  • Manage their medication schedules
  • Book appointments with doctors
  • View medical records
  • Technical expertise: Basic to intermediate

2.2.2 Doctors

  • Healthcare providers using the system
  • Manage patient appointments
  • Create and update medical records
  • Prescribe medications
  • Technical expertise: Intermediate

2.2.3 Administrators

  • System managers with full access
  • Manage user accounts
  • System configuration
  • Generate reports
  • Technical expertise: Advanced

2.3 Operating Environment

  • Client-side: Modern web browsers (Chrome, Firefox, Safari, Edge)
  • Server-side: Windows/Linux servers
  • Database: SQL Server
  • Framework: .NET Core 6+
  • Message Queue: RabbitMQ

3. System Features

3.1 User Authentication and Authorization

FR-001: Secure User Registration

The system shall provide secure user registration functionality

  • Users can register with email and password
  • Email verification required for account activation
  • Password strength requirements enforced
FR-002: Multi-role Authentication

The system shall support multi-role authentication

  • Three user roles: Patient, Doctor, Administrator
  • Role-based access control implementation
  • JWT token-based authentication
FR-003: Password Reset

The system shall provide password reset functionality

  • Forgot password feature via email
  • Secure password reset tokens with expiration
  • Password history tracking

3.2 Patient Management

FR-004: Patient Profile Management

The system shall allow patient profile creation and management

  • Complete demographic information
  • HIV-specific medical data
  • Emergency contact information
  • Unique patient code generation
FR-005: Patient Search

The system shall support patient search functionality

  • Search by patient code, name, or email
  • Advanced filtering options
  • Pagination for large result sets

3.3 Medication Management

FR-007: Medication Database

The system shall provide comprehensive medication database

  • Generic and brand name medications
  • Drug categories and classifications
  • Side effects and contraindications
  • Drug interaction warnings
FR-009: Adherence Tracking

The system shall provide medication adherence tracking

  • Scheduled medication times
  • Actual intake recording
  • Adherence percentage calculation
  • Missed dose tracking

5. Non-Functional Requirements

5.1 Performance Requirements

NFR-001: Response Time
  • Page load time shall not exceed 3 seconds
  • API response time shall not exceed 2 seconds
  • Database query optimization required
NFR-002: Throughput
  • System shall support 1000 concurrent users
  • Database shall handle 10,000 transactions per hour
  • Message queue processing capacity of 1000 messages/minute

5.2 Security Requirements

NFR-007: Data Encryption
  • TLS 1.3 for data in transit
  • AES-256 encryption for data at rest
  • Database encryption at column level
NFR-009: Privacy Protection
  • HIPAA compliance for patient data
  • Data anonymization capabilities
  • Right to data deletion

6. Other Requirements

6.1 Business Rules

BR-001: Patient Registration
  • Unique email address per account
  • Minimum age requirement: 18 years
  • Valid emergency contact required
BR-002: Medication Prescription
  • Only licensed doctors can prescribe medications
  • Prescription expiry dates must be enforced
  • Drug interaction warnings must be displayed

Appendix A: Glossary

Appendix B: Use Case Scenarios

Use Case 1: Patient Medication Adherence Tracking

Actor: Patient
Precondition: Patient is logged in and has prescribed medications

Main Flow:

  1. Patient views today's medication schedule
  2. Patient marks medication as taken
  3. System records adherence data
  4. System updates adherence statistics
  5. System sends confirmation notification

Alternative Flow:

  • If medication is missed, system prompts for reason
  • System sends reminder for missed medication
